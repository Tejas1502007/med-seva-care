HOW IT WORKS:
1. User types message in AARA chat
2. Message sent to /api/chat endpoint
3. Groq API processes with AARA system prompt
4. Response received and displayed
5. Both messages saved to Supabase chat_messages table
6. Messages also stored in localStorage

DATABASE INTEGRATION:
- chat_messages table used for persistence
- Stores: patient_id, role (user/assistant), content, created_at
- Allows doctors to view patient conversations (with consent)
- Enables conversation history across sessions
